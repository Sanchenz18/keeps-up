# 章节2 Git初始化
> “业精于勤，荒于嬉；行成于思，毁于随。”

在之前的章节中我们已经安装了Git，接下来就是相应的用户配置初始化。

## **什么是初始化**
不论是自己的工程需要版本管理，还是与团队其他成员一起协同工作，我们都需要一个自己的身份信息，此时就需要将Git的配置初始化为自己的可以识别的信息。

## **Git用户配置**
需要初始化的信息需要最重要的两个：用户名与邮箱。  
> 配置信息会存放在当前用户根目录下，也就是当前用户只需要配置这一次，往后所有的项目都会优先使用这个全局配置。

```
git config --global user.name your-name
git config --global user.mail your-mail
```
当前用户的所有配置信息可以通过`git config --list`查看。

## **仓库初始化**
对于一个项目，新建文件夹，或者打开已有，此时Git仓库并没有建立，换言之就是Git还不认识这个项目，并不能记录这个文件夹下面的变化。  
所以需要使用`git init`来初始化仓库，将当前的项目下的所有变化存储在这个仓库中。

当完成了初始化，以后每次提交都会看见自己的身份信息，这对于团队协作来说就成为了**可溯源**的贡献。
